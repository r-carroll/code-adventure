# Level 4: Organizing with Methods 📦

As your game gets bigger, the code gets messy. We can use `methods` (like functions) to group our code together.

### Your Task
Ask Gemini:
> "Can you show me how to pass information to a method? For example, a `damage_player(amount)` method that subtracts 'amount' from my HP. Also, explain what a 'Return Value' is and how it can be used to check if the player is still alive."

### What to look for:
- `def name_of_method`: This is how you "define" a new skill.
- `return`: This is how a method sends an answer back.
- `end`: Every method needs an `end` to show where the skill stops.

### Deep Dive:
- **Arguments:** The `(amount)` part is called an argument. It's like giving a specific value to a skill (e.g., "Jump" is the skill, "5 feet" is the argument).
- **Return Values:** It's like a scout returning from a mission with a report. Your code can use that report to decide what to do next.

### Think about this:
- Can you make a method called `heal_player` that *adds* health instead of subtracting it?
- Why do you think programmers use methods instead of just writing one giant list of code? Ask Gemini!
