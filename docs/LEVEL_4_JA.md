# レベル4：メソッドでスッキリさせよう 📦

ゲームが大きくなってくると、コードがごちゃごちゃしてきます。`methods`（メソッド）を使って、コードをグループ分けしましょう。

### タスク
Geminiにこう聞いてみよう：
> "メソッドに情報を『渡す』方法を教えて。例えば、`damage_player(amount)` というメソッドを作って、HPから 'amount' 分だけ減らすようにしたいです。あと、『戻り値（Return Value）』とは何か、それを使ってプレイヤーがまだ生きているか確認する方法も教えて。"

### 注目ポイント：
- `def メソッド名`: 新しい「必殺技（スキル）」を定義（ていぎ）します。
- `return`: メソッドが答えをメインのコードに送り返します。
- `end`: メソッドがどこで終わるかを示すために、必ず最後に `end` が必要です。

### もっと詳しく：
- **引数（ひきすう / Arguments）:** `(amount)` の部分は引数と呼ばれます。スキルに具体的な数字を渡すものです（例：「ジャンプ」がスキルで、「5メートル」が引数）。
- **戻り値（もどりち / Return Values）:** 偵察部隊がミッションから報告書を持って帰るようなものです。その報告を見て、次に何をするか決められます。

- プログラマーは、なぜ1つの長いコードを書かずに、メソッドに分けるんだと思う？ Geminiに理由を聞いてみよう！
