# レベル2：運命の選択 🛣️

ゲームが楽しいのは、自分で選べるからです。コードでは `if`（もし〜なら）と `else`（それ以外なら）を使います。

### タスク
`adventure.rb` を開いて、Geminiにこう聞いてみよう：
> "選択肢を3つにしたいです：'剣'、'盾'、'魔法の杖'。3つ以上の選択肢を扱うための `elsif` の使い方を教えて。あと、ヒアドキュメント（Here Document）を使って、ゲームの始まりにカッコいいアスキーアート（文字の絵）を表示する方法も教えて！"

### 注目ポイント：
- `if / elsif / else`: ゲームの「ルール」を決める命令です。
- `==`: 2つのものが「全く同じか」を確認するときに使います。
- `<<-TEXT`: 長い文章やアスキーアートを表示する「ヒアドキュメント」の始まりです。

### もっと詳しく：
- **elsif:** 「そうでなければ、もし〜なら」という意味です。たくさんの選択肢を順番にチェックできます。
- **比較（ひかく）:** Rubyでは、比べる時に `==`（イコール2つ）を使います。なぜイコール1つ（ `=` ）じゃだめなのか、Geminiに聞いてみよう！

### 考えてみよう：
- 全部大文字で "SWORD" と入力したらどうなるかな？ コンピューターは小文字の "sword" と同じだと気づいてくれるかな？
- Geminiに聞いてみよう：「大文字と小文字を区別しないで、どっちでも動くようにするにはどうすればいい？」
