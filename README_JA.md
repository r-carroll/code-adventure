# コード・アドベンチャー 🚀

ようこそ、冒険者さん！これから、自分だけのテキストアドベンチャーゲームを作ります。そばには、**Ruby**（本物のプログラミング言語です）を少しずつ教えてくれるAI先生がついています。

経験はいりません。ターミナルを使ったことがなくても大丈夫。必要なのは、好奇心だけです。

## ⚡ 一番かんたんな始め方：GitHub Codespaces

自分のパソコンに何もインストールせずに始められる、一番かんたんな方法です。

[![Open in GitHub Codespaces](https://github.com/codespaces/badge.svg)](https://codespaces.new/r-carroll/code-adventure) [![Resume Codespace](https://img.shields.io/badge/Resume_Codespace-238636?style=for-the-badge&logo=github)](https://github.com/codespaces)

1. **初めての人：**「Open in GitHub Codespaces」をクリックします。**続きから遊ぶ人：**「Resume Codespace」をクリックします。（ヒント：このページを開いたままにするには、**Cmd+クリック** か **Ctrl+クリック** を使って新しいタブで開いてね！）
2. 1〜2分待ちます。クラウド上に君の冒険の舞台が準備されます。☁️
3. 準備ができたら、画面の下のほうにある **「ターミナル (Terminal)」** というパネルをクリックします。
4. 次のように入力して、Enterキーを押します。
   ```
   ruby quest.rb
   ```
5. これで冒険のスタートです！🎉

たったこれだけ。この Codespace には、AIの先生（`claude` と `agy`）がもうインストールされていて、いつでも助けてくれる準備ができています。実際に話しかける方法は、下の **AI先生を開いてみよう** を見てね。

## 🧑‍🏫 AI先生を開いてみよう

`ruby quest.rb` は「**何を**作ればいいか」を教えてくれます。でも、実際にコードを書くのを手伝ってくれるのは AI先生です。ここでは、AI先生を開いて話しかける方法を、迷わないように一つずつ説明します。

### ステップ1：もう1つターミナルを開こう

ターミナルを**2つ**並べて使うのがおすすめです。1つはクエストを実行するため、もう1つはAI先生とチャットするためです。

- **GitHub Codespaces（またはVS Code）の場合：** 画面の下にあるターミナルパネルを見てください。その右上にある「**ターミナルを分割**」アイコン（小さな四角が2つ並んだマーク）をクリックします。すると、最初のターミナルの隣に、もう1つターミナルが開きます。
- **Windowsの場合：** Windowsキーを押して `PowerShell` と入力し、Enterを押すと別のウィンドウが開きます。
- **Macの場合：** ターミナルを開いた状態で `Cmd + T` を押すと新しいタブが開きます（`Cmd + N` なら新しいウィンドウです）。

あとからいくつでも増やせるので、心配しなくて大丈夫です。

### ステップ2：AI先生を起こそう

**新しいほうの**ターミナルをクリックして（`ruby quest.rb` を実行しているターミナルはそのままにしておきます）、自分がインストールしたツールに合わせて、次のコマンドを入力してEnterを押します。

| インストールしたツール | 入力するコマンド |
|---|---|
| Claude Code | `claude` |
| Antigravity CLI | `agy` |

少し待ちましょう。カラフルなウェルカム画面が出てきたら、AI先生が起きて、準備完了の合図です。

> **`agy` を初めて使うときは：** Googleアカウントでのサインインを求められます。ブラウザのタブが開くか、（Codespacesの場合は）クリックするためのリンクが表示されます。最初に一度サインインすれば、次からは覚えてくれます。

### ステップ3：質問してみよう

日本語（英語でもOK！）で、そのまま質問を入力してEnterを押します。今のレベルに載っている**パワー・プロンプト**を試してみるのがおすすめです。`ruby quest.rb` を実行するたびに、いくつか表示されます。

### ステップ4：行ったり来たりしよう

チャットを閉じる必要はありません。2つのターミナルを行き来しながら進めましょう。片方でAI先生とチャットして、エディタで `adventure.rb` を編集して、もう片方のターミナルで `ruby quest.rb check` を実行する——これを繰り返せば、レベルアップです！

## 🔁 遊び方の流れ

どのレベルも、同じシンプルな流れで進みます。

1. **実行する** — `ruby quest.rb` を実行すると、物語・やるべきこと・必要な知識が表示されます。
2. **編集する** — `adventure.rb` に自分でコードを書き足します。これが君だけのゲームファイルです。
3. **チェックする** — `ruby quest.rb check` で答え合わせ。何が足りないか、やさしいヒントで教えてくれます（答えそのものは教えません）。
4. **レベルアップ！** ✅ チェックに合格すると、XP（経験値）と新しい称号がもらえて、次のレベルの予告も見られます。

`adventure.rb` という**1つのファイル**を、11レベル通してずっと育てていきます。最後には、全部自分で作ったゲームが完成します。

### コマンド一覧

| コマンド | できること |
|---|---|
| `ruby adventure.rb` | 自分のゲームで遊ぶ！ いつでも実行して、作ったものを試せます |
| `ruby quest.rb` | 進み具合のマップと、今のレベルの物語・やるべきことを表示する |
| `ruby quest.rb check` | 今のレベルの条件に合っているか `adventure.rb` をチェックする |
| `ruby quest.rb map` | 進み具合のマップだけを表示する（クリア済み・現在地・未開放・称号） |
| `ruby quest.rb lang` | 日本語と英語を切り替える |
| `ruby quest.rb reset` | 進み具合をリセットする（`adventure.rb` の中身は消えません） |

## 🐢 自分のペースで進めよう

プログラミングを学ぶのは、楽器の練習や新しいスポーツを覚えるのと同じです。時間がかかって当たり前！

- **焦らなくて大丈夫。** 1日で全部終わらせる必要はありません。
- **1日1レベルでも十分。** 1週間に1レベルでも問題ありません。
- **楽しさを大事に。** もっとやりたければ、どんどん先に進んでOKです！

一番大切なのは、次のレベルに進む前に「**なぜ**このコードが動くのか」をちゃんと理解すること。スピードは関係ありません。

## 🎓 黄金のルール

**AIは君の先生であって、代わりに作ってくれる人ではありません。**

「ゲームを全部作って」とお願いするのはやめましょう。それだとAIは賢くなりますが、**君は賢くなれません。** 君のAI先生はこのルールをすでに知っています（`AGENTS.md`/`CLAUDE.md` に書き込まれています）。もし大きすぎるお願いをすると、やさしく断って、小さな一歩に分けてくれます。

代わりに、こんな**パワー・プロンプト（魔法の質問）**を使ってみましょう。

- 「プレイヤーがリストから武器を選べるようにしたいです。Rubyでどう書きますか？あと、`gets` って実際は何をしているの？」
- 「なんでここで `if` 文を使ったの？他の書き方じゃダメなの？変えたらどうなる？」
- 「エラーが出たけど意味がわかりません。直し方の前に、まずどういう意味か教えてください。」

各レベルの物語画面にも、いくつかパワー・プロンプトのヒントが載っています。

## 💡 困ったときは

行き詰まったり、赤くて怖いエラーメッセージが出たりしたら？

1. **落ち着いて。** エラーは「コンピューターが困っているサイン」なだけです。プロのプログラマーでもよく見ます。
2. **エラーメッセージ全体をコピーしましょう。**
3. **AI先生に聞いてみよう。** エラーを貼り付けて「このエラーはどういう意味？まず何を見ればいい？」と聞いてみましょう。
4. `ruby quest.rb check` はいつでも何度でも実行できます。ヒントを頼りに、自分で答えを見つけてみましょう。
